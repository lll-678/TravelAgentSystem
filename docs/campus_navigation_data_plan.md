# Campus Navigation Data Plan

## Scope

Campus navigation is separate from tourism recommendation:

- recommendation uses real China attractions and schools in `destinations`
- campus navigation uses a selected school's internal map layers and route topology
- current focus is 北京邮电大学沙河校区

## Reference Data Location

Put manually supplied campus files here:

```text
data/reference/bupt-shahe/raw_wgs84/
data/reference/bupt-shahe/topology/
```

Use `raw_wgs84/` for original source layers:

- buildings
- facilities
- roads or paths
- entrances/gates
- other GeoJSON feature layers

Use `topology/` for graph data:

- route nodes
- route edges
- edge geometry
- edge distance/time/mode attributes when available

Use `processed/` only for cleaned intermediate files generated by future scripts.

Downloaded third-party source files go here before cleaning:

```text
data/external/bupt-shahe/osm/
data/external/bupt-shahe/amap_gcj02/
```

`data/external/bupt-shahe/amap_gcj02/` may contain raw AMap GCJ-02 responses. Do not move those raw files into `raw_wgs84/` until they are converted and cleaned.

## Required Coordinate Contract

- WGS84 only.
- Coordinate order is `[lng, lat]`.
- Do not store GCJ-02 source files in this directory.
- Frontend AMap rendering converts WGS84 to GCJ-02.

## Expected Import Flow

1. Read raw JSON/GeoJSON from `data/reference/bupt-shahe/`.
2. Validate geometry and required properties.
3. Normalize categories and names.
4. Deduplicate POIs/buildings.
5. Build or verify road graph connectivity.
6. Snap facilities/buildings/entrances to the graph.
7. Import cleaned rows into `map_nodes`, `map_edges`, `buildings`, `facilities`, and `facility_categories`.
8. Report counts, invalid records, disconnected components, and large snap distances.

## Validation Rules

- Roads/paths must have at least two coordinates.
- Building polygons must have at least three distinct points.
- Facility points must have a name, category, and coordinate.
- Graph edges must reference existing graph nodes or provide endpoint coordinates.
- Edge distances should be present or computable from geometry.
- Facilities with large snap distance should be flagged before route planning.

## Dual POI Datasets

- `nearby_facilities`: full school-surrounding service POIs for nearby facility recommendation.
- `campus_navigation`: BUPT Shahe internal POIs and route endpoint candidates.

Campus navigation imports must apply the BUPT Shahe boundary filter. Nearby facility imports may use a wider radius.

## Do Not

- Do not use raw files directly from API handlers.
- Do not mix these campus files into the nationwide destination recommendation seed.
- Do not commit generated databases or cache files.
- Do not silently convert WGS84 source files to GCJ-02.

## Import Command

The offline import command is implemented:

```bash
PYTHONPATH=backend python backend/scripts/import_reference_campus.py \
  --replace-campus-layers
```

Equivalent admin API source:

```json
{"source": "reference_campus", "reset_existing": true}
```

The command is network-free and covered by backend tests. With the current supplied files, local import produced 106 nodes, 246 road edges, and 35 reference facilities.

Network-enabled source capture commands are documented in `docs/stage_29_dual_poi_sources.md`.
