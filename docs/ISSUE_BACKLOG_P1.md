# P1 Issue Backlog

说明：当前环境没有可用的 `gh`/GitHub Token，所以先把 Issue 清单落成仓库内 backlog。后续可直接按条创建到 GitHub。

## ISSUE-1: 定义小红书内容数据结构

- 目标：统一小红书笔记、地点、标签、图片、亮点、避坑信息的字段。
- 范围：新增/调整后端数据模型与前端 TypeScript 类型。
- 验收：
  - 有稳定的内容对象结构。
  - 能承载本地样例与外部内容两种来源。
  - 不影响现有 `trip plan` 主结构。

## ISSUE-2: 接入小红书内容源服务

- 目标：把小红书内容作为外部信息源接入 POI、推荐和行程理由。
- 范围：后端服务层、必要的配置读取、基础降级逻辑。
- 验收：
  - 可根据城市/关键词获取内容候选。
  - 失败时自动降级到本地样例。
  - 不阻塞 `/api/trips` 主闭环。

## ISSUE-3: 结果页展示内容来源与推荐理由

- 目标：在结果页展示“内容来源”和更可解释的推荐理由。
- 范围：`Result.vue`、相关类型、必要的文案。
- 验收：
  - 用户能看到推荐依据来自哪里。
  - 结果页不新增 demo/fallback 逻辑。
  - 与当前 `trip plan` 数据源一致。

## ISSUE-4: 小红书内容接入测试

- 目标：补齐内容解析、降级逻辑和前端字段显示的最小测试。
- 范围：后端单测、前端类型校验、必要的联调脚本。
- 验收：
  - 内容解析有测试覆盖。
  - 降级链路有测试覆盖。
  - 主闭环测试仍保持通过。

## ISSUE-5: 小红书接入文档与配置收敛

- 目标：把小红书能力写入主文档和启动文档，明确它是 P1 内容增强而非深度抓取主线。
- 范围：`项目总览与执行说明.md`、`docs/README.md`、`docs/QUICK_START.md`。
- 验收：
  - 文档统一口径为“小红书内容接入”。
  - 说明清楚配置项、降级策略和边界。
  - 不与 P0 主闭环描述冲突。
