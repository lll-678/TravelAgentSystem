# 一、项目定位文档

## 项目名称

**Smart Tour Guide：大型校园 / 景区智能导览平台**

## 项目定位

本项目不是简单的课程作业 Demo，而是一个面向大型校园、景区、园区的智能导览平台 MVP。系统目标是为用户提供从“出行前目的地选择”到“游览中导航和服务查询”，再到“游览后游记分享”的完整闭环。

系统主要能力包括：

```text
用户账号体系
目的地推荐
真实地图数据
地图可视化
地点搜索
路线规划
附近设施查询
室内导航
游记社区
游记全文检索
美食推荐
系统管理后台
远程部署
```

虽然课程讲义强调了排序、查找、最短路径、全文检索、压缩等核心算法，但你现在的开发策略可以调整为：**功能优先，算法使用高级实现，必要处保留算法说明和可替换接口**。讲义中的功能要求覆盖旅游推荐、路线规划、场所查询、旅游日记管理、日记交流和美食推荐等模块。 评分原则也要求系统能线下演示全部功能，数据、功能、文档和系统架构要保持一致。

# 二、推荐技术架构

## 技术栈

```text
前端：Vue 3 + Vite + TypeScript + Element Plus / Naive UI
地图展示：高德地图 JS API + @amap/amap-jsapi-loader
后端：FastAPI
数据库：PostgreSQL + PostGIS
缓存：Redis
地图数据与道路拓扑：OpenStreetMap + OSMnx / Overpass
文件存储：本地 media 目录，后续可换 MinIO
搜索：PostgreSQL 全文索引 / pg_trgm，后续可换 Meilisearch
部署：Docker Compose + Nginx
```

## 地图方案边界

```text
后端职责：
- 使用 OSMnx / OpenStreetMap / Overpass 获取真实道路、建筑、设施数据；
- 将道路节点、道路边、建筑轮廓、设施点导入 PostgreSQL + PostGIS；
- 使用 OSM 道路图拓扑完成路线规划和附近设施真实路径距离计算；
- 对前端提供 GeoJSON 或标准经纬度数组。

前端职责：
- 使用高德地图 JS API 负责底图展示、缩放拖拽、Marker、Polygon、Polyline；
- 使用 @amap/amap-jsapi-loader 封装地图 SDK 加载；
- 通过 VITE_AMAP_KEY 读取高德 Web 端 JS API Key；
- 将后端 GeoJSON 坐标转换为高德组件需要的 [longitude, latitude] 数组；
- 不在代码中硬编码高德 Key。

坐标约定：
- 后端 API 默认返回 WGS84 经纬度；
- 前端传给高德覆盖物的坐标统一为 [lng, lat]；
- 默认中心点为北京邮电大学沙河校区。
```

## 架构图

```text
用户浏览器
   ↓
Vue 3 前端
   ↓
Nginx
   ↓
FastAPI 后端 API
   ↓
业务服务层
   ├── 用户服务
   ├── 目的地服务
   ├── 推荐服务
   ├── 地图服务
   ├── 路线规划服务
   ├── 设施查询服务
   ├── 游记服务
   ├── 美食服务
   └── 管理后台服务
   ↓
数据层
   ├── PostgreSQL + PostGIS
   ├── Redis
   ├── OSM 地图数据与道路拓扑
   └── Media 文件存储
```
